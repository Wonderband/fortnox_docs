This table was designed for two purposes simultaneously: (1) a Fortnox sync table — n8n polls GET /3/customers and upserts customer master data from Fortnox's accounting system; and (2) a CRM layer — account management data, financial KPIs, and time tracking per employee. In practice, only the sync function is active. The entire CRM layer (Category C, 46 fields) is empty across both the production base (appk463NGeDy17j9T) and the DEV base (app01yDjf98U6ejKs), confirmed by live queries.

Verified against production base (appk463NGeDy17j9T — Betaportal Adgrabber 2.0): Aktiva Kunder, Bransch, Kundansvarig, MGMT Ansvarig, Säljare, Omsättning, Totalt TB, Totalt fakturabelopp, Fortnox status, Malin tid, Logo, and all other C-category fields return zero records when filtered for non-empty values. The table functions as a pure Fortnox sync table in production.

A — Fortnox-Side Fields  Origin: Fortnox / n8n sync

These fields all originate from the Fortnox accounting system. n8n polls GET /3/customers and upserts records in Airtable using FIELD @url as the upsert key.

⚠️ Any manual edit to an A-field in Airtable will be silently overwritten on the next sync. Changes to customer master data must be made in Fortnox first.

Compared to FortnoxSupplier, the Fortnox Customer API is significantly richer — it includes a full delivery address set, multiple email addresses per document type, invoice/order/offer defaults, and standard document configuration. This reflects Fortnox's role as an invoicing engine: customer records carry all the configuration needed to produce and deliver documents automatically.

Serialized objects: DefaultDeliveryTypes and DefaultTemplates are returned by Fortnox as nested JSON objects (e.g. {"Invoice":"PRINT","Order":"PRINT"}). n8n stores them as JSON strings in singleLineText fields. Similarly, Active and ShowPriceVATIncluded are Fortnox booleans stored as the string "true" or "false" — not Airtable checkboxes.

C — CRM Fields (Designed, Never Used)  Empty in both bases · intended for staff use

These 46 fields were designed as Adgrabber's own customer management layer — not from Fortnox, not written by n8n. They were intended for account managers to track responsibility, financial KPIs, and time worked per customer.

All 46 C-fields are empty in both production and DEV bases. Verified via live queries on production (appk463NGeDy17j9T).
