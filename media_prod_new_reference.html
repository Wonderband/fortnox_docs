TABLE Media / Prod New — The Campaign Line Item Join Point

Base: app01yDjf98U6ejKs (DEV. Betaportal Adgrabber 2.0 Feb 2026)  ·  Table ID: tblTf9sNLy464khq5

Role in the system. Each record in this table = one campaign line item — a specific media channel (or production task) for a specific campaign. It is the central join point between customer revenue (via sys.orderallocationmapping) and supplier cost (via sys.fakturamapping). All P&L calculations for a campaign channel happen on or aggregate to this record.

Two Record Types: Mediakop vs Production

FIELD Sync Source (singleSelect) determines the type of line item. This controls which budget formula branches are active.

Sync Source valueWhat it representsRevenue source
Dev Sync Mediakop Media buy — Adgrabber places ads on behalf of the customer (Google Ads, Facebook, Programmatic, etc.) Brief. Kundens mediabudget — the customer order amount, manually set
Dev Sync Production Production work — creative production, consulting, etc. Ek. budget. Intäkt_Prod — manually set production revenue budget

Budget Field Hierarchy — Full Classification

None of the budget fields are written by any sys table. sys.orderallocationmapping and sys.fakturamapping both link TO this table; they read from it (via lookup fields) but never write back. All budget inputs originate here, set manually by staff at campaign setup.

Layer 1 — Type-Specific Revenue Derivation formula

Ek. budget. Intäkt_Media

IF(Brief. Använder kundens konto? (Avser Google, FB/IG, LinkedIn),
   Brief. Kundens mediabudget − Brief. Budget mediaspend,
   Brief. Kundens mediabudget)

Customer uses own account (true): customer pays Brief. Budget mediaspend directly to the platform — that amount is already gone from their budget. Adgrabber earns only the remainder (management fee). Result: Ek. budget. Intäkt_Media = Brief. Kundens mediabudget − Brief. Budget mediaspend.

Adgrabber manages account (false, standard): Brief. Budget mediaspend does not apply. Adgrabber receives the full customer budget as revenue. For M8347: Ek. budget. Intäkt_Media = 9,600 SEK.

Layer 2 — Canonical Revenue Field formula

Total Ek. budget. Intäkt — THE authoritative revenue figure for all P&L calculations.

IF(Sync Source = "Dev Sync Mediakop",
   Ek. budget. Intäkt_Media,
   Ek. budget. Intäkt_Prod)

Selects the right revenue figure based on line type. All margin and budget check formulas reference this field. Value for M8347: 9,600 SEK.

Layer 2b — UI Display Field formula

Budget — shown in record detail views.

IF(Sync Source = "Dev Sync Mediakop",
   Brief. Kundens mediabudget,
   Total Ek. budget. Intäkt)

For media lines: returns the gross customer order amount — the number the customer recognises as their spend commitment.
For production lines: same as Total Ek. budget. Intäkt.
Difference from Total Ek. budget. Intäkt: only when customer uses own account (then Budget = gross, Total Ek. = net management fee).
Sys. M Mediabudget or P Productionsbudget is an exact duplicate of this formula — legacy field.

Where Budget Values Come From — Data Flow

Budget values flow into Media/Prod New from staff input, and out to the sys tables as lookups. The sys tables never write back. The allocation table reads Brief. Kundens mediabudget via a rollup field for display, but this is one-way.
1
Campaign agreement. Customer commits to paying e.g. 9,600 SEK for Google Ads Q4. Staff creates Media/Prod New record, sets Brief. Kundens mediabudget = 9,600 SEK manually. Formulas Total Ek. budget. Intäkt and Budget resolve immediately.
2
Orders confirmed and synced. n8n syncs Fortnox orders as FortnoxOrder + Fortnox OrderRow records. The order row totals (3,200 + 3,072 + 3,072 = 9,344 SEK for M8347) may differ slightly from the planned budget — this is normal.
3
Staff creates allocation records. For each order row staff manually creates two sys.orderallocationmapping records (one per allocation type), entering FIELD ALLOCATION Brief. Customer Budget (How much we allocate) — the SEK amount for that month and type. These amounts are derived from the order row total (not the planned budget). Media/Prod New then shows the sum as rollup fields (ALLOCATED MEDIA BUDGET_Rollup, ALLOCATED to Spend on Mediakop, etc.).
4
Supplier invoices arrive. Staff creates sys.fakturamapping records entering Belopp ex moms — the actual invoiced cost. Media/Prod New shows the sum as Ek. utfall. Externkostnad (Fakturamapping) (V5). This is actual cost tracking, not budget setting.
5
Accrual check. The difference between allocated spend (ALLOCATED to Spend on Mediakop = 7,476 SEK) and actual cost (7,709 SEK) = UPPLUPNA = +233 SEK. This does NOT indicate a budget overrun — the revenue budget (Total Ek. budget. Intäkt = 9,600 SEK) is not exceeded.

Allocation Integrity Fields

These fields check whether the planned budget has been fully allocated in the sys tables, and whether actual costs match the plan.

FieldTypeWhat it checks
ALLOCATED MEDIA BUDGET_Rollup rollup Sum of ALL allocation amounts (both types) from sys.orderallocationmapping for this line. M8347: 9,344 SEK.
ALLOCATED to Spend on Mediakop rollup Sum of "Pay for mediaköp" type allocations only — the pre-allocated platform spend. M8347: 7,476 SEK.
Tutti All Allocated_media rollup Total allocated revenue for media lines. M8347: 9,344 SEK.
Brief. Kundens mediabudget Difference To Allocation formula Tutti All Allocated_media − Brief. Kundens mediabudget. Negative = not all customer budget has been allocated. M8347: −256 SEK 🔴 (orders totalled 9,344 vs planned 9,600).
Ek. utfall. Externkostnad (Fakturamapping) (V5) rollup Sum of Belopp ex moms from all linked sys.fakturamapping records. Actual supplier cost. M8347: 7,709 SEK.
UPPLUPNA (Supplier cost less Alloc Spend) (V5) formula Actual cost − Allocated spend. Positive = accrued cost exceeds planned spend estimate. M8347: +233 SEK.
