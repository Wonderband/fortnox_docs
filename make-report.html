Make Implementation Report

Invoice automation workflow

Concise report of the current Make.com implementation that starts from Gmail, extracts invoice data with OpenAI, uploads the PDF to Google Drive, writes a row into Google Sheets, applies a processed label, and sends a confirmation email.

Status: Working end-to-end Platform: Make.com LLM: OpenAI Responses API Storage: Google Drive + Google Sheets

1. Current flow

  1. Gmail watches incoming emails.
  2. Attachments are listed and filtered to PDF only.
  3. PDF is uploaded to OpenAI via /v1/files.
  4. Extraction request is sent to /v1/responses.
  5. Returned JSON text is parsed into structured fields.
  6. Original PDF is uploaded to Google Drive.
  7. A share link is created for the uploaded PDF.
  8. A new row is added to Google Sheets.
  9. The Gmail label invoice-processed is added.
  10. A custom confirmation email is sent.

2. Implemented nodes

  • Gmail → Watch emails
  • Gmail → List email attachments and media
  • Filter → PDF only
  • HTTP → Upload file to OpenAI
  • HTTP → Extract invoice fields from OpenAI
  • JSON → Parse JSON
  • Google Drive → Upload a File
  • Google Drive → Get a Share Link
  • Google Sheets → Add a Row
  • Gmail → Modify Email Labels
  • Gmail → Send an email

3. Data written to sheet

  • Supplier
  • Description
  • Receipt item
  • Amount
  • Date
  • Comment
  • PDF link

4. Credentials and auth

  • OpenAI API key stored in Make HTTP auth.
  • Google Drive required custom Google Cloud OAuth.
  • Google Sheets can reuse the same Google Cloud project/client.
  • Restricted Google scopes were added manually.

5. Important scopes

  • .../auth/drive
  • .../auth/drive.readonly
  • .../auth/spreadsheets
  • Gmail was connected separately in Make.

6. Main implementation challenges and resolutions

  • OpenAI request design: two HTTP requests were used: upload PDF first, then call the extraction endpoint with the returned file_id. This was simpler in Make than injecting base64 into one request.
  • Incomplete model responses: low max_output_tokens caused incomplete outputs. Increasing the token budget fixed this.
  • OpenAI output structure: the response returned both reasoning and message items. The final JSON lived in the message branch.
  • JSON parsing in Make: parsing initially failed because the wrong array element was referenced. The working mapping targeted the final message text explicitly.
  • Google Drive connection: personal Gmail required a custom Google Cloud OAuth app instead of the default restricted connection flow.
  • Drive scope errors: 403 insufficient authentication scopes was resolved by adding Drive scopes in Google Cloud and reconnecting the Make connection.
  • Google Sheets row insertion: Make failed on a non-native spreadsheet file. The destination had to be a real Google Sheet, not an Excel file in Drive.
  • Date display in Sheets: 2026-04-10 appeared as 46122, which is a normal Sheets date serial value. Formatting the column as Date fixes the display.

7. What is already working

  • Email trigger works.
  • PDF filtering works.
  • OpenAI returns valid structured JSON.
  • PDF uploads to Google Drive.
  • Share link is generated.
  • New row is added to Google Sheets.
  • Processed label is added in Gmail.
  • Confirmation email is sent.

8. Recommended next improvements

  • Add a dedicated error route with failure email + processed label.
  • Restore stricter schema enforcement if needed.
  • Improve Drive filenames using invoice date + supplier.
  • Create year/month folders automatically instead of using a fixed folder.
  • Pre-format the date column in Sheets as YYYY-MM-DD.
  • Tighten Gmail filters to avoid non-invoice PDFs.

9. Summary

The Make implementation is currently working on the main success path. It starts from Gmail, processes PDF attachments through OpenAI, parses the returned JSON, uploads the original PDF to Google Drive, writes the extracted data into Google Sheets, adds a processed label, and sends a custom confirmation email.

The main technical effort went into OpenAI response handling inside Make, Google restricted OAuth setup for Drive/Sheets, and resolving spreadsheet/document-format issues.
