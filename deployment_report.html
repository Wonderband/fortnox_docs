Lessons Learned During Setup

⚠️ N8N crash loop on startup Old data/ folder from a local run had a SQLite DB encrypted with a different key. Fix: wipe data/ contents before first server start.
⚠️ N8N_SECURE_COOKIE env var not forwarded Variables in .env.server are NOT automatically passed to the container. They must also be listed explicitly in the environment: block of docker-compose.yml.
ℹ️ Fortnox OAuth2 required ngrok HTTPS Fortnox only accepts HTTPS redirect URIs. n8n generates the OAuth callback URL from the browser address bar — so you must open n8n via an ngrok HTTPS URL during OAuth setup. After OAuth completes, restore WEBHOOK_URL to the plain HTTP server address.
