6. Authentication Flow

6.1 Standard OAuth2 (Authorization Code)

  1. n8n (automation.offertizer.com) initiates OAuth2 Authorization Code flow
  2. User / service account grants permission at id.fortnox.se
  3. Fortnox redirects back to:
       https://automation.offertizer.com/rest/oauth2-credential/callback
     with ?code={authorization_code}
  4. n8n exchanges code for access_token + refresh_token
  5. n8n stores tokens in its credential store
  6. API calls: GET/POST/PUT https://api.fortnox.se/3/{endpoint}
       Header: Authorization: Bearer {access_token}
  7. Access tokens expire after 1 hour — n8n auto-refreshes using refresh_token

6.2 Service Account / Client Credentials (new, Dec 2025)

  Both integration-adgrabber and adgrabber-websocket2 have "Service Account" enabled.

  New Client Credentials flow (available since Dec 3, 2025):
  - No refresh tokens needed
  - Suitable for server-to-server workflows with no human login
  - POST https://api.fortnox.se/oauth-v1/token
      grant_type=client_credentials
      client_id=y9FK6cDsK8N1
      client_secret={secret}
  - Returns access_token directly
  - Recommended migration path to simplify token management in n8n

8. Important Notes & Action Items

8.1 API URL migration (completed Feb 9, 2026)

Fortnox deprecated apps.fortnox.se/3 on Feb 9, 2026. All calls now redirect to api.fortnox.se/3. If any n8n nodes still use the old URL, they should be updated — though redirects are currently in place.

8.2 Integration inactive warning

integration-adgrabber is flagged as Inactive in Fortnox (45+ days unused as of 2026-03-12). If n8n workflows stop working, try the "Activate integration" button in Fortnox → Menu → Integrations → My integrations. The OAuth tokens themselves appear valid (token exchange chart shows recent Mar 2026 activity).

8.3 Client Credentials migration opportunity

Both integrations have Service Account mode enabled. Fortnox now supports the Client Credentials OAuth2 grant type (released Dec 3, 2025) — this eliminates refresh token management. Consider migrating the n8n Fortnox credential to Client Credentials for simpler, more reliable auth.

8.4 Webhook support (new)

Fortnox now supports webhooks for: integration purchase, rating/review, license expiry, and user invitations/deactivations. Currently not configured for Adgrabber. Could replace or supplement polling-based n8n workflows for near-real-time sync.
