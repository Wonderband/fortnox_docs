TABLE FortnoxInvoice — Full Field Audit

Table ID: tblOLaZyIaVJP6h6V  |  Base: Betaportal Adgrabber 2.0 (appk463NGeDy17j9T)  |  Total fields: 108  |  Audit date: 2026-03-26  |  Data sample: 30 records (prod)

FortnoxInvoice is the customer-side invoice table. n8n polls GET /3/invoices and upserts records using @url as the upsert key. These are invoices Adgrabber sends to its clients — they are generated in Fortnox from orders and flow into Airtable as read-only mirrors. The table sits at the end of the revenue chain: FortnoxOrder → FortnoxInvoice → (payment). There is no Airtable → Fortnox push direction for this table.

Key architecture note — two order linkage fields: FortnoxInvoice links to FortnoxOrder via two separate mechanisms: (1) FIELD FortnoxOrder — a direct multipleRecordLinks field pointing to tblhNdAzxbWeNgEgW, populated by n8n during sync (prefersSingleRecordLink: true — enforced 1:1 in the UI); and (2) FIELD OrderReference — a plain text field containing the Fortnox order DocumentNumber (e.g. "2670"), copied directly from the Fortnox API payload. The link field is the authoritative Airtable connection; OrderReference is the raw Fortnox value. This table also has a child-record table: FortnoxInvoiceRow (tblDkNRTKMTpMTLK3) — individual line items parsed from the InvoiceRows JSON blob, linked via FIELD FortnoxInvoiceRow.

Why 94 Fortnox-side fields? The Fortnox Invoice API object is unusually large — it bundles customer details, delivery address, payment terms, accounting codes, VAT breakdowns, boolean flags, and extended metadata into one payload. n8n maps every field from the payload into Airtable regardless of whether the data is meaningful for Adgrabber's use case. Of the 94 fields, only 49 carry active data; 33 are either always-zero constants or always empty.

A — Fortnox-Side Fields  Origin: Fortnox / n8n sync

n8n polls GET /3/invoices and upserts records using @url as the upsert key. Manual edits to A-fields in Airtable will be overwritten on the next sync. All invoice data changes must be made in Fortnox.

Boolean fields: Fortnox returns booleans as JSON values but n8n stores them as the string "true" or "false" in Airtable singleSelect or multilineText fields. InvoiceRows is the full invoice line-item array serialized as a JSON string. EDIInformation and EmailInformation are also stored as JSON strings.
