3. Fortnox Integration

How it works: n8n (or equivalent) polls the Fortnox REST API and syncs data into dedicated Airtable tables. Each table has a *_Sync view that n8n uses as a filter, and control fields like Internal Status (Do not sync / Sync) to enable/disable sync per supplier.

3.1 Direction of data flow

Table Direction Trigger / Control
FortnoxCustomer Fortnox → Airtable n8n sync via FortnoxCustomer_Sync view
FortnoxSupplier Fortnox → Airtable Internal Status field: "Do not sync" / "Sync" — controls per-supplier
FortnoxSupplierInvoice Fortnox → Airtable Only pulled for suppliers where Internal Status = Sync. Also: "Platform - Not Fortnox" records from ad platform CSV imports.
FortnoxOrder Airtable → Fortnox (bidirectional) Drafted in Airtable → Sync Fortnox = "To sync" triggers n8n to create in Fortnox → Fortnox writes back DocumentNumber
Fortnox OrderRow Both LineItem Source = "Fortnox" or "Airtable users"
FortnoxInvoice Fortnox → Airtable n8n sync, linked to FortnoxOrder by DocumentNumber
FortnoxInvoiceRow Fortnox → Airtable Rows per invoice, linked to FortnoxInvoice and FortnoxArticle
FortnoxArticle Fortnox → Airtable Product/article catalog. Linked to sys.mediatyp for media channel classification

4. Supplier Invoice Mapping & Accruals (Upplupna)

This is the critical financial reconciliation workflow connecting incoming supplier invoices to campaign line items. 

  STEP 1: Supplier invoice arrives in Fortnox
     ↓
  STEP 2: n8n syncs it to FortnoxSupplierInvoice (if supplier has Sync=enabled)
     ↓
  STEP 3: Staff creates sys.fakturamapping record(s):
          FortnoxSupplierInvoice ──► Media/Prod New line item(s)
          with field: Belopp ex moms (SEK amount allocated to this M/P)
     ↓
  STEP 4: Media/Prod New rolls up the costs:
          - Ek. utfall. Externkostnad = actual supplier cost received
          - UPPLUPNA = allocated spend (budgeted) minus actual invoiced cost
          - Emoji formula shows mapping status: 🟢 🔴 ⚪ ⚠️
     ↓
  STEP 5: Automation Upplupna table
          - n8n automation runs monthly accrual calculations
          - Stores: Upplupna Automation Found Allocations Invoiced,
            Found Mapping Cost, Run Date, Modified Date
Table Role in accruals
sys.fakturamapping Bridge table: links one FortnoxSupplierInvoice to one or many Media/Prod New records. Staff manually links these. Tracks: Belopp ex moms, Allocated External Costs (V5), Variance (±100 SEK tolerance).
FortnoxSupplierInvoice Source: actual supplier cost. Rollup from fakturamapping shows "Ek. utfall. Externkostnad". Formula "Invoice fully mapped?" shows reconciliation status.
Media / Prod New Destination: actual costs surface here as rollups. UPPLUPNA = budgeted spend − invoiced cost. Automation Upplupna records track monthly automation runs.
Automation Upplupna Stores results of automated accrual processing runs. Fields: run date, found allocations invoiced, mapping cost, supplier GBG amounts.

Platform invoices (non-Fortnox path)

Some suppliers (Google, Facebook/Meta, LinkedIn etc.) are marked Supplier type = "Platform - Not Fortnox" in FortnoxSupplier. Their cost data comes from Platform Import Files (CSV/report uploads) rather than Fortnox API. FortnoxSupplierInvoice records are created for these too, but sourced from the platform file, not Fortnox. The rest of the mapping flow (sys.fakturamapping → Media/Prod New) is identical.

5. Order Allocation Flow (sys.orderallocationmapping)

  FortnoxOrder
      └──► Fortnox OrderRow (line items)
               └──► sys.orderallocationmapping (allocation table)
                        ├──► Media/Prod New [Media]       (what media buy it covers)
                        ├──► Media/Prod New [Production]  (what production work it covers)
                        └──► FortnoxSupplier              (which supplier is being paid)

Each order row can be allocated across multiple Media/Prod New records. Fields track:

  • ALLOCATED Budget (how much of the order row budget is assigned)
  • ALLOCATED to Spend (how much the supplier will be paid)
  • Outstanding (Not Allocated) Order budget
  • Type of allocation (from sys.Type of allocation lookup)
  • Upplupna flag — whether this allocation is an accrual
